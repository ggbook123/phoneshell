# 多端连接方案（本地引导 + 客户端主动连 Relay）

**目标**
- 手机在群组中时，通过扫码把同一局域网内的客户端拉入群组。
- 客户端主动连接 Relay，Relay 统一维护群组与设备列表并广播同步。
- 设备列表支持离线显示，重连后自动更新。

**前提条件**
1. 手机已连接 Relay 并完成绑定（bind）。
2. 手机与被邀请客户端在同一局域网，手机可访问客户端 `http://<client-ip>:<port>/api/invite`。
3. 客户端提供 `/api/invite` 接口并支持从 standalone 切换到 client 模式。
4. Relay 支持 `invite.create.request` 与 `group.join.request(inviteCode)`。

**端到端流程（时序）**
1. 手机已在群组内。
2. 手机扫码客户端二维码，得到 `httpUrl` 与 `deviceId`。
3. 手机向 Relay 发送 `invite.create.request`，拿到 `inviteCode + relayUrl + expiresAt`。
4. 手机向客户端 `POST /api/invite`，下发 `relayUrl + inviteCode + groupId`。
5. 客户端收到后切换到 client 模式，主动连接 Relay。
6. 客户端发送 `group.join.request(inviteCode)`。
7. Relay 验证邀请码，更新群组与设备列表，持久化后广播：
   - `group.member.list`
   - `group.member.joined`
   - `device.list`
8. 所有端（手机、网页、其它设备）设备列表实时更新。

**接口定义**
- 手机 → Relay 申请邀请码
  - 请求：`invite.create.request`
  - 响应示例：
```
{
  "type": "invite.create.response",
  "inviteCode": "ABCD1234",
  "relayUrl": "wss://relay.example.com/ws/",
  "expiresAt": "2026-03-15T12:00:00Z"
}
```

- 手机 → 客户端下发邀请
  - `POST /api/invite`
```
{
  "relayUrl": "wss://relay.example.com/ws/",
  "inviteCode": "ABCD1234",
  "groupId": "G123",
  "expiresAt": "2026-03-15T12:00:00Z"
}
```

- 客户端 → Relay 加入群组
```
{
  "type": "group.join.request",
  "inviteCode": "ABCD1234",
  "deviceId": "pc-001",
  "displayName": "Office-PC",
  "os": "Windows",
  "availableShells": ["powershell", "cmd"]
}
```

**设备列表同步策略**
1. Relay 以群组成员为权威来源，合并在线设备状态。
2. `device.list` 包含离线成员，`isOnline=false`。
3. 设备加入/退出/改名/踢出时，Relay 广播 `device.list`。

**持久化策略**
1. Relay 端持久化群组信息（group.json）。
2. 设备端持久化群组密钥（group-membership.json）用于断线重连。

**安全约束**
1. 邀请码一次性、短期有效，避免泄露长期 `groupSecret`。
2. `groupSecret` 仅在邀请成功加入后回传给设备，用于后续重连。

**限制与说明**
- 如果手机与客户端不在同一局域网，手机无法直连客户端 `/api/invite`。
- 跨网段场景需要改用“客户端主动拉取邀请”或“常驻连接”等方案。

**验收点**
1. 手机扫码客户端后，客户端自动加入群组。
2. Relay 更新群组与设备列表并广播。
3. 所有端设备列表实时同步，离线设备仍可见。
