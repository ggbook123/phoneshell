1. 实现场景

场景描述：用户在办公室/家里通过电脑运行 PowerShell（PS）。下班后，用户可以通过手机（iOS、安卓、鸿蒙）远程接管、查看并继续操作 PS 会话。每台电脑仅能绑定一个手机，但一个手机可以绑定多台电脑。通过二维码扫描绑定，手机端可以看到登录状态，并具有“强制中断控制权”的功能。

2. 实现目标

目标一：实现 PC 和手机端的双向控制，确保手机端能实时查看、控制电脑上正在运行的 PowerShell 会话。

目标二：PC 端要显示二维码，手机端扫描后才能绑定，确保一台手机只能绑定一台电脑。

目标三：手机端只能绑定并控制一台电脑，且手机与电脑之间的控制权限由二维码认证与2FA（双因素认证）进行严格控制。

目标四：PC 端能够看到手机端登录，并且能够通过按钮强制中断手机端控制权，将控制权交给PC端。

3. 完整架构设计
3.1 电脑端（PC端）

功能模块：

会话托管服务（Session Manager）：

管理多个 PowerShell 会话，可以启动、控制和销毁 PS 会话。

将 PS 会话的输入和输出转发给连接的客户端（手机端和 PC 客户端）。

支持多个客户端连接，但每个会话只允许一个活动控制者（手机端或 PC 端）。

客户端连接模块：

生成二维码，供手机端扫描进行绑定。

控制权限管理，确保一个手机端只能连接一台电脑。

管理连接的手机端显示其当前状态，并允许 PC 端通过按钮强行中断控制权。

UI 界面：

显示二维码，用于绑定手机端。

显示所有连接状态，显示当前由哪个设备控制。

提供“强制中断控制”的按钮，允许管理员从电脑端接管控制。

技术栈：

语言：C# 或 Python（C# 更适合 Windows 环境，Python 更通用）

框架：

.NET Core（C#）用于服务端会话管理。

Windows Terminal 用于本地 PS 窗口显示和输出。

WebSocket / gRPC：用于双向通信（PC <-> 手机端）。

QRCode Generator：用于生成二维码。

3.2 手机端（iOS、安卓、鸿蒙端）

功能模块：

二维码扫描：手机端通过扫描电脑端生成的二维码与电脑绑定，确保手机端只能绑定到一台电脑。

客户端连接：使用 WebSocket 或 gRPC 实时与电脑端的 Session Manager 进行通信。

输出显示：手机端接收到电脑端的 PowerShell 输出，并进行实时显示。支持ANSI颜色、进度条等显示。

输入控制：手机端通过触摸屏输入命令并发送到会话池中的 PowerShell 会话。支持普通输入、命令粘贴、快捷键等。

控制权切换：手机端可以主动请求控制，并在电脑端看到切换请求，管理员可以强制中断手机端控制。

技术栈：

iOS（Swift）：使用 Swift 和 WebSocket 连接，UI 使用 UIKit 或 SwiftUI 进行设计。

安卓（Kotlin/Java）：使用 Kotlin（或 Java）和 WebSocket 进行连接，UI 使用 Android SDK。

鸿蒙（C）：使用 C/C++ 与鸿蒙提供的通信接口进行连接，UI 使用鸿蒙 SDK（HarmonyOS）。

3.3 服务器端

功能模块：

用户管理：提供手机端和电脑端的登录认证与绑定，手机与电脑之间的一对一绑定。

权限控制：确保每次只能有一个设备拥有控制权（手机端优先）。提供认证机制（如二维码扫描和2FA）。

消息队列：传递控制权请求、会话输出、输入等信息。

技术栈：

Node.js / .NET Core（可以选择与主应用语言一致）

WebSocket / gRPC：用于客户端与服务器的实时双向通信。

4. 功能模块
4.1 电脑端功能

二维码生成：

当电脑端启动时，会自动生成一个唯一的二维码，用户手机扫描二维码绑定设备。

生成的二维码包含设备的唯一标识和认证信息。

手机端认证与绑定：

手机端扫描二维码后，通过 2FA 认证完成绑定。

每台手机只允许绑定一台电脑，二维码绑定后手机端的控制权限才会生效。

会话管理：

当手机端连接时，会话管理模块会分配一个新的 PS 会话并开始转发。

每个会话的输入输出会通过 WebSocket 或 gRPC 转发到连接的设备。

控制权管理：

电脑端提供一个“强制中断控制权”的按钮，PC 用户可以通过该按钮断开手机端的连接，重新获取控制权。

4.2 手机端功能

二维码扫描与认证：

手机端扫描电脑端生成的二维码并完成绑定，通过 2FA 完成认证后开始连接。

终端显示与输入：

手机端显示从电脑端发送过来的 PS 输出，并支持触摸屏输入（支持粘贴、行编辑、回显等）。

支持全屏终端显示，用户可以在手机端控制 PowerShell 会话。

控制权管理：

手机端可以请求获取控制权并显示请求信息，PC 端可以看到并操作“强制中断”按钮。

手机端控制权限优先，电脑端只能查看输出而无法输入，除非控制权被重新切换回 PC。

6. 补充功能（可选）

会话日志记录：记录每次会话的历史日志（输入、输出、操作等），以便后续审计。

断线重连：当手机端掉线时，自动恢复连接并继续显示输出。

多用户管理：管理员可以管理多个设备的控制权限，可以将某个手机端的权限禁用或重置。
