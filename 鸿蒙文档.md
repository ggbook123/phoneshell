**连接与状态管理（DeviceConnection）**

来源：`entry/src/main/ets/services/DeviceConnection.ets`

- 基于 WebSocket。
- 自动重连：指数退避，基准 1000ms，最多 10 次，最大间隔 30s。
- `ConnectionState`：`DISCONNECTED(0)` / `CONNECTING(1)` / `CONNECTED(2)`。
- 支持消息回调与状态回调注册/注销。

**连接与状态管理（ConnectionManager）**

来源：`entry/src/main/ets/services/ConnectionManager.ets`

- 模式：`STANDALONE`、`SINGLE`、`GROUP`。
- 单连设备：`singleConnections` + `singleDeviceInfos` + `singleDeviceHttpUrls`。
- 群组连接：`groupConnection`，维护 `groupDevices` / `groupMembers` / `groupId` / `groupSecret`。
- 会话管理：`sessions` Map，维护 `activeSessionId` / `activeDeviceId`。

关键行为

- 启动时恢复连接：优先恢复群组连接，其次恢复单连设备。
- 单连设备连接成功后会请求 `device.list` 以补齐设备信息。
- 单连设备被指定为 relay 时，提升为群组连接并立即 `group.join.request`。
- 群组连接重连成功后会自动发送 `group.join.request`。
- `getConnectionForDevice` 优先命中单连设备，否则回落到群组连接。

**授权管理（AuthManager）**

来源：`entry/src/main/ets/services/AuthManager.ets`

- 维护待审批队列（5 分钟 TTL）。
- 收到 `auth.request` 立即触发 UI 弹窗显示。
- `approve/reject` 会发送 `auth.response`。
- `AuthManager.showPending()` 用于弹出队列中的下一条请求。

**二维码格式与扫码逻辑**

二维码格式

- 单连：`phoneshell://connect?http=...&ws=...&deviceId=...&displayName=...`
- 群组绑定：`phoneshell://bind?server=...&groupId=...&groupSecret=...&serverDeviceId=...`
- 面板登录：`phoneshell://login?server=...&groupId=...&requestId=...`

扫码逻辑（DeviceManagePage）

- 扫码使用 `ScanKit`，`enableAlbum=true`，`enableMultiMode=false`。
- 若二维码格式不以 `phoneshell://` 开头，提示“无效二维码格式”。

单连连接

- 若未入群：直接 `connectSingle(ws, deviceId, displayName, httpUrl)`。
- 若已入群：不直接连接，弹出邀请确认框，通过 HTTP 邀请该设备加入群组。

群组绑定

- 已入群且群组相同：提示“已在该群组中”。
- 已入群且群组不同：从 `server` 转换为 `http` 地址，发起邀请加入当前群组。
- 未入群：建立群组连接 -> `group.join.request` -> 成功后 `mobile.bind.request` 并拉取设备列表。

面板登录

- 若手机未在群组中：提示“重新连接群组”，等待 `group.join.accepted` 后发送 `panel.login.scan`。
- 若已在群组中：直接发送 `panel.login.scan`。
- 处理 `error` 消息：`not_bound_mobile` / `login_session_not_found` / 其他错误。

- `fitAndReport(true)` 首次触发 `onTerminalReady(cols, rows)`，后续 resize 触发 `onTerminalResize`。

**关键流程（时序摘要）**

应用启动

1. `LoginPage` 初始化偏好与语言。
2. 生成或读取 `mobileDeviceId`。
3. 初始化 `AuthManager`。
4. 恢复群组/单连连接。

扫码连接单设备

1. 扫描 `phoneshell://connect`。
2. 未入群：`connectSingle` -> 建立 WebSocket。
3. 已入群：弹出邀请确认 -> HTTP `POST /api/invite`。

扫码群组绑定

1. 扫描 `phoneshell://bind`。
2. 未入群：`connectGroup` -> `group.join.request` -> `mobile.bind.request`。
3. 已入群且群组不同：转换 HTTP 地址 -> 邀请加入当前群组。

扫码面板登录

1. 扫描 `phoneshell://login`。
2. 已入群：直接 `panel.login.scan`。
3. 未入群：等待 `group.join.accepted` 后发送 `panel.login.scan`。

会话与终端

1. SessionList 发送 `terminal.open`。
2. 收到 `terminal.opened` 进入 `TerminalPage`。
3. WebView ready -> `terminal.resize` + `terminal.history.request`。
4. 输入/快捷键 -> `terminal.input`。

**Flutter 复刻注意点**

- 全局配色、卡片/标签风格是视觉一致性的关键。
- 终端体验依赖 WebView + xterm.js，需保持 JS Bridge API 一致。
- 设备与群组状态需与服务端消息保持严格同步。
- 扫码错误提示与状态文案需完整呈现。
- 弹窗层级与遮罩颜色需要统一（`#B0000000`）。
