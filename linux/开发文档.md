# PhoneShell Linux2 — Node.js + TypeScript 重构开发文档

## 一、项目概述

将现有 C# (.NET 8) Linux 服务端完全用 Node.js + TypeScript 重写。
保持与 HarmonyOS 手机端的 WebSocket 协议完全兼容，前端从内嵌 C# 字符串改为独立 Vue 3 + TypeScript 项目。

### 替代关系

| 旧 (C#) | 新 (TypeScript) |
|---------|----------------|
| `PhoneShell.Core/Networking/RelayServer.cs` (~900行) | `src/relay/relay-server.ts` |
| `PhoneShell.Core/Protocol/Messages.cs` + `MessageSerializer.cs` | `src/protocol/messages.ts` + `serializer.ts` |
| `PhoneShell.Core/Networking/WebPanelModule.cs` + `WebPanelHtml.cs` (~1600行) | `web/` Vue 3 独立前端 |
| `PhoneShell.Core/Terminals/Linux/PtySession.cs` + `libptyhelper.so` | `src/terminal/pty-session.ts` (用 node-pty) |
| `PhoneShell.Headless/HeadlessHost.cs` (~960行) | `src/server/app.ts` |
| `PhoneShell.Headless/HeadlessConfig.cs` | `src/config/config.ts` |
| `PhoneShell.Headless/Program.cs` | `src/index.ts` |
| `PhoneShell.Core/Services/GroupStore.cs` | `src/store/group-store.ts` |
| `PhoneShell.Core/Services/DeviceIdentityStore.cs` | `src/store/device-store.ts` |
| `PhoneShell.Core/Models/GroupInfo.cs` | `src/protocol/messages.ts` (统一类型) |
| `PhoneShell.Core/Models/DeviceIdentity.cs` | `src/protocol/messages.ts` (统一类型) |

## 三、协议兼容性（关键约束）

新项目**必须**与现有 HarmonyOS 手机端 WebSocket 协议完全兼容。
所有消息类型、字段名、JSON 格式保持一致。

### 消息类型清单 (30种)

**设备管理：**
- `device.register` — 设备注册 {deviceId, displayName, os, availableShells}
- `device.unregister` — 设备注销
- `device.list.request` / `device.list` — 设备列表

**会话管理：**
- `session.list.request` / `session.list` — 会话列表

**终端操作：**
- `terminal.open` — 打开终端 {deviceId, shellId}
- `terminal.opened` — 终端已打开 {deviceId, sessionId, cols, rows}
- `terminal.input` — 终端输入 {deviceId, sessionId, data}
- `terminal.output` — 终端输出 {deviceId, sessionId, data}
- `terminal.resize` — 终端调整大小 {deviceId, sessionId, cols, rows}
- `terminal.close` / `terminal.closed` — 关闭终端

**控制权：**
- `control.request` / `control.grant` / `control.force_disconnect`

**群组管理：**
- `group.join.request` / `group.join.accepted` / `group.join.rejected`
- `group.member.joined` / `group.member.left` / `group.member.list`
- `group.kick`

**手机绑定：**
- `mobile.bind.request` / `mobile.bind.accepted` / `mobile.bind.rejected`
- `mobile.unbind`

**授权：**
- `auth.request` / `auth.response`
- `panel.login.scan`

**服务器迁移：**
- `group.server.change.request` / `group.server.change.prepare` / `group.server.change.commit`

**密钥轮换：**
- `group.secret.rotate.request` / `group.secret.rotate.done`

**错误：**
- `error` — {code, message}

### JSON 序列化规则
- 字段名使用 camelCase
- null 值不输出 (WhenWritingNull)
- type 字段用 `.` 分隔的字符串

## 四、REST API 兼容性

### 公开端点 (无需认证)
- `GET /panel` → 返回前端 HTML
- `GET /panel/` → 同上
- `GET /panel/xterm.min.js` → xterm.js 库
- `GET /panel/xterm.min.css` → xterm.js 样式
- `GET /panel/addon-fit.min.js` → xterm.js fit 插件
- `GET /api/panel/verify` → 验证 token 是否有效
- `GET /api/panel/pairing` → 获取配对信息 (wsUrl, groupId, hasBoundMobile)
- `GET /api/panel/qr.png` → 配对二维码 PNG
- `GET /api/panel/login/start` → 开始登录流程
- `GET /api/panel/login/status/{requestId}` → 查询登录状态
- `GET /api/panel/login/qr.png?payload=...` → 登录二维码 PNG

### 需认证端点 (Bearer token)
- `GET /api/status` → 服务器状态
- `GET /api/devices` → 设备列表
- `GET /api/group` → 群组信息
- `GET /api/sessions/{deviceId}` → 设备会话列表

### WebSocket 端点
- `ws://host:port/ws/` → WebSocket 连接（支持 query param `token` 认证）

### 健康检查
- `GET /ws/healthz` → 健康检查
- `GET /ws/status` → 服务器运行状态

## 六、与旧项目已修复 Bug 的对应处理

### Bug 1: 手机断开后 panel token 被清除
- **旧代码位置**: `RelayServer.cs` 第533-539行
- **处理方案**: `relay-server.ts` 中 mobile disconnect 时只清 `panelLoginSessions`，不清 `panelAccessTokens`

### Bug 2: fork() 导致 SIGSEGV
- **旧代码位置**: `PtySession.cs` + `libptyhelper.so`
- **处理方案**: 直接使用 `node-pty`，完全不存在此问题

### Bug 3: 重新进入终端显示空白
- **旧代码位置**: `HeadlessHost.cs` RawOutputBuffer + `RelayServer.cs` snapshot 发送
- **处理方案**: `terminal-manager.ts` 中 `OutputBuffer` 类 + 重新订阅时发送 snapshot

## 七、部署方案

### 目录结构
```
/opt/phoneshell/
├── node_modules/        # 依赖
├── dist/                # 编译后的 JS
├── web/                 # 前端静态文件
├── package.json
└── ...
```

### 配置文件 (兼容旧路径)
- `/etc/phoneshell/config.json` — 服务配置
- `/etc/phoneshell/data/device.json` — 设备身份
- `/etc/phoneshell/data/group.json` — 群组数据

### systemd 服务
```ini
[Unit]
Description=PhoneShell Server
After=network-online.target

[Service]
Type=simple
WorkingDirectory=/opt/phoneshell
ExecStart=/usr/bin/node dist/index.js --config /etc/phoneshell/config.json
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### nginx 配置
保持现有 nginx 配置不变，只需确保 upstream 端口一致 (19090)。

## 八、详细开发任务

### 阶段 1: 项目骨架 + 协议层
1. 初始化 Node.js 项目 (package.json, tsconfig.json)
2. 定义所有消息类型 (`src/protocol/messages.ts`)
3. 实现消息序列化器 (`src/protocol/serializer.ts`)
4. 实现配置加载 (`src/config/config.ts`)
5. 实现数据存储 (`src/store/device-store.ts`, `group-store.ts`)

### 阶段 2: 终端管理
6. 封装 node-pty (`src/terminal/pty-session.ts`)
7. 实现输出缓冲 (`src/terminal/output-buffer.ts`)
8. 实现终端生命周期管理器 (`src/terminal/terminal-manager.ts`)

### 阶段 3: WebSocket 中继服务器
9. 实现客户端连接抽象 (`src/relay/client-connection.ts`)
10. 实现中继服务器核心 (`src/relay/relay-server.ts`)
    - 设备注册/注销
    - 群组加入/管理
    - 手机绑定
    - 终端消息转发
    - Panel 认证流程
    - 健康检查

### 阶段 4: HTTP 服务 + 认证
11. 实现 token 管理 (`src/auth/token-manager.ts`)
12. 实现二维码服务 (`src/auth/qr-service.ts`)
13. 实现 HTTP 服务 + REST API (`src/server/app.ts`)
14. 实现入口文件 (`src/index.ts`)

### 阶段 5: 前端 (Vue 3)
15. 初始化 Vue 3 + Vite 项目
16. 实现 WebSocket 客户端 (自动重连 + 心跳)
17. 实现二维码登录页
18. 实现设备列表/仪表盘页
19. 实现终端视图 (xterm.js)
20. 打包为单页面

### 阶段 6: 集成 + 部署
21. 主入口集成所有模块
22. 编译 + 部署配置
23. Bug 检查 + 修复
24. 编写开发完成报告
